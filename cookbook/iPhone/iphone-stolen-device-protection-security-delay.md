# iPhone 显示安全延迟，无法立即更改 Apple 账户或关闭失窃设备保护

“安全延迟”通常不是故障，而是 iPhone 的“失窃设备保护”在保护 Apple 账户和关键设置。最快路径是确认顾客要做的操作、是否在熟悉地点，以及是否能完成 Face ID 或 Touch ID；多数情况下需要等待一小时后再次完成生物识别认证。

---

## 症状

- “iPhone 提示安全延迟，要等一小时。”
- “我在改 Apple ID 密码，为什么不能马上改？”
- “想关闭失窃设备保护，但一直要求等。”
- “人在家里也显示安全延迟。”
- “回收、换机、折抵时卡在安全延迟。”
- “Face ID 坏了，能不能用密码跳过？”
- “安全延迟是不是账号被盗了？”

---

## 可能原因

1. **失窃设备保护已开启**
   - Apple 说明，当 iPhone 离开家或办公室等熟悉地点时，某些敏感操作需要额外安全要求。
2. **敏感操作触发一小时等待**
   - Apple 说明，更改 Apple 账户密码等安全相关操作需要等待一小时，然后再次进行 Face ID 或 Touch ID 认证。
3. **生物识别不能用设备密码替代**
   - Apple 说明，失窃设备保护要求 Face ID 或 Touch ID，不能使用设备密码作为备选认证方式。
4. **熟悉地点不是手动立即指定的白名单**
   - 社区讨论反复出现“我在家也延迟”的困惑。不要承诺添加家庭地址后可立即绕过延迟；这会削弱该功能的保护目的。
5. **交易、折抵或送修时才发现延迟**
   - Apple 的出售、赠送或折抵指南提醒，如果启用了失窃设备保护，在商店更改密码可能会有一小时安全延迟。

---

## Apple 官方方案

验证级别：Apple 官方

步骤：

1. 先确认顾客正在做什么：更改 Apple 账户密码、关闭失窃设备保护、改设备密码、关闭“查找”、抹掉设备、访问已存储密码，还是锁定 App。
2. 如果 iPhone 显示安全延迟，向顾客说明这是失窃设备保护的正常安全要求，不要把它当作系统卡死。
3. 按屏幕提示完成第一次 Face ID 或 Touch ID 认证。
4. 让 iPhone 保持可用，等待一小时。不要反复改定位、重启、退出登录或抹掉设备来尝试绕过。
5. 一小时后回到同一项操作，再完成第二次 Face ID 或 Touch ID 认证。
6. 如果操作目的是更改 Apple 账户密码，按 Apple 的更改 Apple 账户密码流程继续。
7. 如果操作目的是出售、赠送、折抵或送修，完成安全延迟后再关闭失窃设备保护、关闭“查找”或抹掉设备。
8. 如果 Face ID 或 Touch ID 无法工作，先修复生物识别问题或联系 Apple 支持；失窃设备保护下的相关操作不能简单用设备密码替代。
9. 如果 iPhone 已丢失或失窃，优先在 iCloud.com 或另一台受信任设备上使用“查找”标记为丢失，并保护 Apple 账户。
10. 如果 Apple 账户密码也忘记，按 Apple 账户密码重设或账户恢复路径处理；安全延迟和账户恢复等待时间是两个不同问题。

## 哪些操作可能触发

验证级别：Apple 官方

Apple 明确把安全延迟和关键安全设置关联起来，例如更改 Apple 账户密码。Apple 还说明，失窃设备保护会让关键设备和 Apple 账户操作需要 Face ID 或 Touch ID，并在更敏感的操作上加入等待。

常见顾客场景：

1. 更改 Apple 账户密码。
2. 关闭失窃设备保护。
3. 准备出售、赠送、折抵或抹掉 iPhone。
4. 访问锁定 App、已存储密码、付款或账户相关敏感信息时被要求生物识别。
5. 在陌生地点处理账户或安全设置。

---

## 什么时候可以避免现场卡住

验证级别：Apple 官方

1. 准备折抵、出售、赠送或送修前，提前在常用地点处理 Apple 账户密码、“查找”、失窃设备保护和抹掉设备。
2. 到店前确认 Face ID 或 Touch ID 能正常使用。
3. 不要临时更改 Apple 账户密码后立刻开始换机、折抵或维修流程。
4. 如果已经触发安全延迟，按屏幕提示等待并完成第二次生物识别认证。
5. 如果顾客无法等待，让顾客知道这是账户安全机制，不是门店或支持人员可以跳过的等待。

## 已验证的非官方处理思路

非官方

### 反复切换定位、VPN 或“重要地点”

- 来源：Apple Support Community 中有用户讨论定位服务、VPN、重要地点和熟悉地点。
- 成功概率：未知。
- 风险：中等。可能造成更多定位和账户状态混乱，也不能保证取消安全延迟。
- 备注：Apple 官方说明安全延迟用于防止他人重大更改账户或设备；不要把调整定位当作官方绕过方法。
- 验证级别：未知

### 为折抵或二手交易提前关闭

- 来源：Reddit 和 Apple 社区反复出现“到店才发现要等一小时”的提醒；Apple 出售、赠送或折抵指南也明确提醒可能有安全延迟。
- 成功概率：高，前提是顾客仍控制设备、Face ID 或 Touch ID 正常，并且提前安排时间。
- 风险：低到中等。关闭保护会降低失窃场景下的账户防护，完成交易准备后不要长期裸奔使用。
- 备注：这是规划建议，不是绕过安全延迟。
- 验证级别：较可能

## 零售排查流程

1. 先问顾客屏幕原文和当前操作，不要直接建议抹掉或重启。
2. 解释一句核心原因：“这是失窃设备保护，目的是防止别人拿到你的手机和密码后立刻改账户。”
3. 确认 Face ID 或 Touch ID 是否能完成。如果不能，先转生物识别或硬件排查。
4. 如果已触发安全延迟，记录开始时间，让顾客等待一小时后回来完成第二次认证。
5. 如果顾客正在折抵、出售、赠送或维修，优先安排等待后的关闭“查找”、关闭保护和抹掉设备流程。
6. 如果顾客担心账号被盗，检查是否是本人刚刚发起的操作；如果不是本人发起，立即帮助其保护 Apple 账户并标记丢失设备。
7. 不推荐通过关闭定位、改家庭地址、VPN、重启或抹掉设备来绕过等待。
8. 处理完成后建议顾客保留失窃设备保护，除非正在完成转让、折抵或维修交接。

---

## 升级处理

联系 Apple 支持：

- Face ID 或 Touch ID 失效，导致无法完成失窃设备保护要求。
- 顾客没有发起操作却看到安全延迟，怀疑账户或设备被他人控制。
- Apple 账户密码忘记，需要账户恢复。

前往 Apple Store 或授权维修点：

- TrueDepth 摄像头、Touch ID、显示屏或触控硬件异常。
- 设备需要维修，但无法完成交接前的账户和查找设置。

不要升级为维修：

- 单纯的一小时安全延迟通常不是硬件故障。
- 单纯不想等待，不是可绕过的维修问题。
